# Security Policy

Report vulnerabilities privately through the repository's security advisory flow. Do not include credentials, customer data, or task transcripts in an issue.

The dispatcher is a local policy and audit tool, not a sandbox or authorization boundary. Review hook commands before trusting them, keep Codex and Node current, and use managed mode only when you administer the target machine.

Version 2.0 deliberately allows a dispatcher-launched worker with the explicit `--git` / `git_access: true` capability to run all Git commands and GitHub CLI repository operations directly with the user's existing credentials when they are within the bounded task. This includes potentially irreversible actions such as force pushes, tags, branch deletion, and destructive `gh` repository operations. Git access is not an authorization workflow: it needs no grants, allowlists, release plans, acknowledgement digests, or second confirmations. It is rejected for read-only routes and uses `danger-full-access` only for that dispatched Git task; ordinary write tasks remain `workspace-write`. Repository content and task text are untrusted input: task injection can direct a worker to make irreversible remote changes. Sol remains blocked from direct Git and GitHub mutations. Restrict credentials, repository permissions, and worker task scope; inspect untrusted repositories before dispatching work. The CLI fallback remains tied to an owner-only installer manifest plus a regular owned wrapper's recorded SHA-256, rather than a filename or suffix; symlinks, modified wrappers, malformed manifests, and `PATH` hijacks fail closed. It does not make local Git, hooks, credentials, or the user's account a security boundary. Audit storage and readers use a strict field/type allowlist: legacy artifact paths are migrated to stable SHA-256 values under `artifact_path_sha256`, raw artifact paths and nested data are never returned, malformed lines expose only a constant marker, and raw error text is reduced to a safe code plus SHA-256 provenance.
